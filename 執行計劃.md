# SLOG 化療護理幫手 - 執行計劃

> 專為 SLOG 雞尾酒療法設計的照護輔助應用程式，支援療程追蹤、照護須知、用藥紀錄與家族備忘功能。

---

## 一、專案概述

### 1.1 目標用戶
- 主要：接受 SLOG 化療的患者本人
- 次要：家屬（共同照護、備忘協作）

### 1.2 技術規格
| 項目 | 規格 |
|------|------|
| 前端框架 | React / Next.js（支援 SSR、適合 Vercel 部署） |
| 樣式 | Tailwind CSS，手機版優先（Mobile-first） |
| 資料庫 | Vercel Postgres 或 Supabase（支援即時更新、多人協作） |
| 部署 | Vercel |
| 認證 | 可選：簡單密碼/邀請碼，或 Vercel Auth |

### 1.3 應用程式架構（參考截圖）

```
┌─────────────────────────────────────┐
│  [圖示] 化療護理小幫手               │  ← App Header
├─────────────────────────────────────┤
│                                     │

│  - 療程進度追蹤 / 照護須知           │

│                                     │
├─────────────────────────────────────┤

└─────────────────────────────────────┘
```

## 二、功能模組詳細規格

---

### 2.1 行事曆（療程進度追蹤）

#### 2.1.1 頁面結構
- **頂部標題**：療程進度追蹤（含行事曆圖示）
- **進度概覽區**：顯示「目前進度 (共 N 次)」、當前「第 X 次療程」「第 Y 天」
- **當日提示卡片**：依日期自動顯示該日應做事項與注意事項
- **14 天週期視覺化**：Day 1～Day 14 的網格/按鈕，可點選查看各日說明
- **底部操作**：重新設定日期

#### 2.1.2 療程階段定義（對應治療計劃.md）

| 天數 | 階段 | 系統顯示重點 |
|------|------|--------------|
| **Day 1** | 打針日 | 醫院打針、攜帶冰敷手套/腳套、當晚開始口服藥 |
| **Day 2-7** | 口服藥週 | TS-1 + Leucovorin、預防藥、速養療、副作用觀察 |
| **Day 8-14** | 休養週 | 停藥、免疫力最低期、體力恢復、Day 14 抽血準備 |

#### 2.1.3 功能細節

| 功能 | 說明 |
|------|------|
| **設定起始日** | 使用者選擇「打化療針劑日」為該次療程 Day 1 |
| **設定療程次數** | 可設定總療程數（預設 6 次，依病情總結） |
| **自動計算** | 依起始日 + 14 天週期，自動計算當前為第幾次、第幾天 |
| **當日提示** | 依 Day 1～14 自動顯示：打針提醒、口服藥提醒、休養注意、抽血提醒等 |
| **14 天視覺區分** | Day 1 / Day 2-7 / Day 8-10 / Day 11-14 以不同樣式區分（不參考截圖顏色） |
| **重新設定日期** | 若療程延後或調整，可重新設定起始日 |

#### 2.1.4 資料結構（資料庫）

```typescript
// 療程設定
interface TreatmentConfig {
  id: string;
  startDate: string;        // 第一次療程 Day 1 的日期 (YYYY-MM-DD)
  totalCycles: number;      // 總療程次數 (預設 6)
  cycleLength: number;      // 每週期天數 (固定 14)
  updatedAt: string;
}
```

### 2.2 照護須知

#### 2.2.1 頁面結構
- **標題**：照護須知
- **分類卡片**：依主題分區塊呈現，可摺疊/展開
- **內容來源**：病情總結.md、治療計劃.md 彙整 + 可查證的醫學資訊

#### 2.2.2 內容分類（依治療計劃.md 彙整）

| 分類 | 內容重點 |
|------|----------|
| **飲食與營養** | 絕對熟食、高蛋白飲食、少量多餐、避免冰冷（草酸鉑） |
| **副作用防護** | 嘴破/腹瀉預防（速養療、軟毛牙刷、漱口）、手腳麻預防（冰敷手套、保暖） |
| **感染預防** | 早晚量體溫、勤洗手戴口罩、家人感冒時隔離 |
| **中醫配合** | 肝腎功能正常前提、榮總傳統醫學部、勿用偏方 |
| **急診警訊** | 發燒 38°C+畏寒、嚴重嘔吐/水瀉、劇烈腹痛/呼吸困難/意識模糊 |

#### 2.2.3 功能細節

| 功能 | 說明 |
|------|------|
| **靜態內容** | 由專案內 Markdown/JSON 或 CMS 管理，可後續更新 |
| **搜尋** | 可選：關鍵字搜尋照護內容 |
| **書籤** | 可選：使用者標記常用段落 |

#### 2.2.4 資料結構

```typescript
// 照護須知（可存於 CMS 或 JSON）
interface CareGuideline {
  id: string;
  category: string;         // 飲食、副作用、感染、中醫、急診
  title: string;
  content: string;         // Markdown 或 HTML
  order: number;
}
```

### 2.3 用藥清單（每日用藥紀錄與比對）

#### 2.3.1 參考用藥指南.ts 的功能架構

| 用藥指南既有功能 | 用藥清單頁面對應 |
|------------------|------------------|
| 每日課表（早/中/晚/睡前） | 每日用藥紀錄：依時段顯示，可打勾紀錄 |
| 備用藥品（PRN） | 有症狀才吃：拉肚子、過敏等情境 |
| 藥品圖鑑 | 圖文對照、可上傳自訂藥丸照片 |

#### 2.3.2 頁面結構
- **Tab 1：每日課表**  
  - 依療程 Day 動態顯示：Day 1～7 顯示完整口服藥；Day 8～14 僅顯示非化療藥（若有）
  - 早/中/晚/睡前時段分組
  - 每顆藥可打勾紀錄，支援「重置今日打勾」
  - 與行事曆連動：當日若為打針日，顯示打針提醒
- **Tab 2：備用藥品**  
  - 依症狀分類（拉肚子、過敏等）
  - 顯示劑量、時機
- **Tab 3：藥品圖鑑**  
  - 藥丸外觀描述、可上傳/更換照片

#### 2.3.3 用藥清單「比對」功能（新增）

| 功能 | 說明 |
|------|------|
| **每日完成率** | 計算當日應吃藥項 vs 已打勾數，顯示完成百分比 |
| **遺漏提醒** | 若到睡前仍有未打勾項目，可顯示提醒（可選推播） |
| **歷史紀錄** | 可查詢過去某日的用藥紀錄，供回診時參考 |
| **與療程連動** | Day 1～7 顯示 TS-1/Folina；Day 8～14 自動隱藏或標示「本週停藥」 |

#### 2.3.4 資料結構（資料庫）

```typescript
// 用藥課表（可來自用藥指南.ts 的 scheduleData）
interface MedicationSchedule {
  id: string;
  timeOfDay: string;       // 早上/中午/晚上/睡前
  timing: string;          // 早餐前/早餐中/早餐後...
  meds: MedicationItem[];
}

interface MedicationItem {
  id: string;
  baseId: string;
  name: string;
  dose: string;
  purpose: string;
  applicableDays?: number[];  // 1-7 表示 Day 1~7 才顯示，空則每天
}

// 每日用藥紀錄
interface DailyMedRecord {
  id: string;
  date: string;            // YYYY-MM-DD
  userId: string;
  checkedItems: Record<string, boolean>;  // medId -> 是否已吃
  updatedAt: string;
}
```

### 2.4 備忘錄（家族協作）

#### 2.4.1 頁面結構
- **標題**：備忘錄
- **卡片列表**：每則備忘以卡片形式呈現
- **新增按鈕**：浮動或固定位置，點擊新增卡片
- **卡片操作**：每張卡片可編輯、刪除

#### 2.4.2 功能細節

| 功能 | 說明 |
|------|------|
| **新增** | 標題 + 內容（可簡短），以卡片形式加入列表 |
| **刪除** | 單張卡片可刪除，需確認 |
| **修改** | 點擊卡片進入編輯模式，可改標題與內容 |
| **排序** | 可選：拖曳排序或依時間排序 |
| **多人使用** | 家族成員透過同一連結/帳號進入，所見即所得 |
| **即時同步** | 使用資料庫 + 即時訂閱（如 Supabase Realtime），一人修改他人即時看到 |

#### 2.4.3 資料結構（資料庫）

interface MemoCard {

## 三、資料庫設計總覽

### 3.1 資料表

| 表名 | 用途 |
|------|------|
| `treatment_config` | 療程設定（起始日、總次數） |
| `daily_med_records` | 每日用藥打勾紀錄 |
| `memo_cards` | 備忘錄卡片 |
| `pill_images` | 可選：藥丸自訂照片（或存於 Storage） |

### 3.2 部署考量
- **Vercel Postgres**：輕量、與 Vercel 整合佳
- **Supabase**：支援 Realtime、Auth、Storage，適合多人協作
- 若初期無後端，可先用 **localStorage** 開發，再遷移至資料庫

## 四、開發階段建議

### Phase 1：核心架構（1–2 週）
1. Next.js 專案建立，Tailwind 設定
2. 底部導覽列、路由結構
3. 行事曆頁面：療程進度、14 天週期、當日提示（先用前端 state，無 DB）
4. 照護須知：靜態內容呈現

### Phase 2：用藥與資料持久化（1–2 週）
1. 用藥清單頁面：移植用藥指南.ts 邏輯，加入療程連動
2. 每日完成率、比對邏輯
3. 資料庫選型與建立（Supabase / Vercel Postgres）
4. 療程設定、用藥紀錄寫入 DB

### Phase 3：備忘錄與多人協作（約 1 週）
1. 備忘錄 CRUD 頁面
2. 即時同步（若用 Supabase）
3. 簡單分享機制（連結/邀請碼）

### Phase 4：優化與部署（約 1 週）
1. 手機版 UI 優化、PWA 支援（可選）
2. Vercel 部署、環境變數設定
3. 基本錯誤處理與載入狀態

## 六、備註

- **不參考截圖的顏色與風格**：僅採用架構與資訊層級，視覺由專案設計決定
- **SLOG 專用**：療程邏輯針對 14 天週期、6 次療程設計，若未來擴充其他療法需抽象化
- **隱私**：備忘錄與用藥紀錄涉及健康資料，部署時需考量存取權限與資料保護
