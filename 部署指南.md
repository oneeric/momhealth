## 部署步驟

### 方式一：透過 Vercel 網頁（推薦）

1. **登入 Vercel**
   - 前往 [vercel.com](https://vercel.com)
   - 使用 GitHub / GitLab / Bitbucket 登入

2. **匯入專案**
   - 點擊「Add New」→「Project」
   - 選擇「Import Git Repository」
   - 若尚未連接，先將專案推送到 GitHub

3. **設定專案**
   - Framework Preset：**Next.js**（自動偵測）
   - Root Directory：`./`（預設）
   - Build Command：`npm run build`（預設）
   - Output Directory：`.next`（預設）
   - **環境變數**：需設定 Upstash Redis（見「雲端共享設定」）

4. **部署**
   - 點擊「Deploy」
   - 等待建置完成（約 1～2 分鐘）
   - 完成後會得到一個網址，例如：`https://momhealth-xxx.vercel.app`

## 注意事項

### 雲端共享設定（Upstash Redis）

所有使用者會看到同一份療程、用藥紀錄與備忘。需在 Vercel 專案中設定 Upstash：

1. 前往 [Vercel Marketplace - Upstash](https://vercel.com/integrations/upstash)
2. 點擊「Add Integration」→ 選擇專案 `momhealth`
3. 建立新的 Redis 資料庫（或選擇既有）
4. 完成後，`UPSTASH_REDIS_REST_URL`、`UPSTASH_REDIS_REST_TOKEN` 會自動注入
5. 重新部署專案使環境變數生效

**本地開發**：複製 `.env.example` 為 `.env.local`，從 [Upstash Console](https://console.upstash.com/) 取得連線資訊填入。

### LINE 提醒（選用）

1. 在 Vercel 環境變數新增：
   - `LINE_CHANNEL_ACCESS_TOKEN`
   - `LINE_CHANNEL_SECRET`
   - `CRON_SECRET`（建議，用於保護 Cron 端點，可用 `openssl rand -hex 32` 產生）
2. 在 LINE Developers（Messaging API）設定 Webhook URL：
   - `https://你的網域/api/line/webhook`
3. 開啟 **Use webhook** 並驗證成功
4. 將官方帳號加好友並傳一則訊息，系統會自動綁定推播對象
5. 在 App「設定」頁可看到目前綁定狀態

### 資料儲存
- 有設定 Upstash 時：資料存於 **Redis**，所有使用者共享
- 未設定時：fallback 到 **localStorage**，僅本機可見

### 藥品圖片
- `public/pills/` 與 `public/pills/preview/` 會一併部署
- 請確認圖片已加入 Git 並推送

### 自訂網域（選用）
- 在 Vercel 專案 → Settings → Domains 可綁定自己的網域
